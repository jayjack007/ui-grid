Modal usega Help :

The WNP Modal window is different from opening a new browser window. When the Modal window is open the main window can not be accessed. Other difference is that the Modal window content are loaded when the page loads, while when a new browser window is opened that content is loaded when the link is clicked.

Feel free to use the two predefined Modal window ("large", and "small").
See below how you would normaly use a Modal window with WNP. 

      <wnp-pop url="views/modalHelp.html" wnp-pop-name="large">  
      . . .
Right click here will open the Modal Window 
      . . .
      </wnp-pop>

Usually, you need to provide two values, those are the URL of the Modal window content, and a name of the Modal window.
We predefined two Modal windows you can use. The one is you can see above, and this is the one is used by this Modal window. It's called "large". The other one is called "small". You can see the name is refering to the Modal window size.

If those two windows are not enough, you can create your own. Look at the Bootstrap Modal documentation for more information. http://getbootstrap.com/javascript/#modals .
You can predefine Modal window by modifying 'WindowsPopupConfig.js' file.

When you create a new Modal window there are some rules that you need to follow. See below how this Modal window is predefined in 'WindowsPopupConfig.js' file: 

    var largeModalElement = '<!-- Large modal -->' +
  '<div class="$$$modalName$$$ modal fade bs-example-modal-lg" tabindex="-1" role="dialog" aria-labelledby="mySmallModalLabel" aria-hidden="false">' +
    '<div class="modal-dialog modal-lg">' +
      '<div class="wnp-modal modal-content" > ' +
      '<button type="button" class="close" data-dismiss="modal"><span aria-hidden="true">×</span><span class="sr-only">Close</span></button>' +
        '<div ng-include="' + "'" + '$$$modalContentUrl$$$' + "'"  + '">' +
          '<!-- Modal content -->' +
        '</div>' +
      '</div>' +
    '</div>' +
  '</div>';
There are two placeholders you MUST have in your Modal DOM element. The windowsPopup module will replace those to the proper values.
  • $$$modalName$$$ --> This is needed to add there a unique name that identifies that Modal. This unique name is used to popup Modal window. Also unique name is needed to make sure that only one "Modal Element" is added to the page. You can have many "wnp-pop" directive refering to the same URL and same wnp-pop-name on the same page. At the first occurance the "Model Element" is added, but not at the subsequence ones. The unique name is created by the url and the wnp-pop-name. In the above example the unique name would be viewsmodalHelphtmllarge . See that the slash and period characters are removed.
  • $$$modalContentUrl$$$ --> This is the URL that will be pulled in to the page at the time when the page is loaded. Note, when you do the right click the content already loaded and no network access is needed. That is why this Modal display faster to display then when the same URL is opened by the wnp-popup directive, where a new browser window is opened.
